Feature Branch Workflow


The core idea behind the Feature Branch Workflow is that all feature development should take place in a dedicated branch instead of the master branch. This encapsulation makes it easy for multiple developers to work on a particular feature without disturbing the main codebase. Instead of committing directly on their local master branch, developers create a new branch every time they start work on a new feature. Feature branches should have descriptive names, like scrollview-menu-items or issue-#1061. The idea is to give a clear, highly-focused purpose to each branch.

We will be using Github's Issues in order to keep track of our project.

Create a Branch to fix the issue

When you want to work on a new feature or fix a bug you make a new branch based on the tip of master. You would
git checkout master
git pull origin master
git checkout -b make-sliders-blue-#27 master

Closing issues in a commit

The following keywords will close an issue via commit message:
close closes closed
fix fixes fixed
resolve resolves resolved
For example you might do a commit like:
git add -A
git commit -m "This closes #27, sliders are now blue"
git push -u origin make-sliders-blue-#27

Pull Requests

Aside from isolating feature development, branches make it possible to discuss changes via pull requests. Once someone completes a feature, they donâ€™t immediately merge it into master. Instead, they push the feature branch to the central server and file a pull request asking to merge their additions into master. This gives other developers an opportunity to review the changes before they become a part of the main codebase.

Merge Branch & Resolve Issues


Once a pull request is accepted and you are ready to push it to master you can merge by checking out master
git checkout master
do a pull to get the latest version
git pull
merge in your feature by pulling the feature-branch
git pull origin make-sliders-blue-#27
git push
and you now have merged in the feature. If that feature is done and that branch wont be needed again you can clean up by deleting the branch. You can delete a local branch by running
git branch -d feature-x
or a remote branch by running
git push origin --delete feature-x
or, the less verbose but harder to remember
git push origin :feature-x
